# openpassivehub-catalog

Independent public catalog of earning apps, tools, links, addons, and
related resources. Broader in scope than the M4B-specific repos: lists
everything — whether or not it integrates with
[Money4Band](https://github.com/MRColorR/money4band) — and provides a
GitHub Pages site with weighted referral-link rotation.

## Relationship to the other catalogs

| Repo | Role |
|------|------|
| [`MRColorR/money4band`](https://github.com/MRColorR/money4band) | Core CLI + built-in ToS-compliant supported apps |
| [`MRColorR/money4band-community-catalog`](https://github.com/MRColorR/money4band-community-catalog) | Community apps + addons (M4B-runnable, opt-in) |
| **`MRColorR/openpassivehub-catalog`** (this repo) | Full ecosystem index (apps, tools, VPNs, captcha-solvers, etc.) |

## Layout

This repo uses the **same per-file layout** as the two M4B repos so
entries can be promoted/demoted across repos by plain file rename:

```
catalog/
├── supported/          # Established, reliable entries
├── newcomers/          # Recent additions / under evaluation
└── deprecated/         # Dead, scammy, or retired — kept for reference
addons/                 # Helpers / sidecars (mirrors community-catalog)
manifest.json           # Aggregated wire format, regenerated by CI
scripts/
└── build_manifest.py   # Shared with money4band core (keep in sync)
site/                   # GitHub Pages static site (weekly rebuild)
```

Every bundle in `catalog/` follows the same M4B app-bundle schema
documented in
[`money4band/docs/app-management.md`](https://github.com/MRColorR/money4band/blob/main/docs/app-management.md#app-bundle-schema),
plus an optional `integration` field marking entries that do **not**
run under M4B (external-only):

```jsonc
{
  "name": "SOME_EXTERNAL_APP",
  "integration": "external-only",   // or "m4b" (default)
  "compose_config": null,           // allowed for external-only
  ...
}
```

## Using this catalog from M4B

Although this repo is informational, M4B can consume any entry whose
`integration` is `"m4b"` by adding a remote source:

```jsonc
{
  "type": "remote",
  "url":  "https://raw.githubusercontent.com/MRColorR/openpassivehub-catalog/main/manifest.json",
  "default_source": "openpassivehub",
  "cache_path": "catalog/openpassivehub",
  "enabled": false,
  "timeout": 5.0
}
```

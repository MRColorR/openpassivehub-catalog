# Contributing to openpassivehub-catalog

Thanks for helping grow the openpassivehub index!

## TL;DR

1. Add or edit one file under `catalog/<category>/<NAME>.json` (or
   `addons/<NAME>.json`) per pull request.
2. Follow the M4B app-bundle schema documented in
   [`money4band/docs/app-management.md`](https://github.com/MRColorR/money4band/blob/main/docs/app-management.md#app-bundle-schema).
3. Filename stem = `name` field (uppercase, `[A-Z][A-Z0-9_]+`).
4. Set `"integration": "m4b"` for Money4Band-runnable entries;
   `"integration": "external-only"` for anything else (VPNs,
   captcha-solvers, external earning apps without Docker support, etc.).
5. `manifest.json` is regenerated by CI — do **not** edit by hand.

## Categories

| Directory | Meaning |
|-----------|---------|
| `catalog/newcomers/` | New additions; under community evaluation |
| `catalog/supported/` | Established entries with verified payouts |
| `catalog/deprecated/` | Dead / scammy / retired — with `additional_details` |
| `addons/` | Helpers that attach to a parent entry |

## Scope

This repo indexes the **whole** passive-income ecosystem, including
non-M4B tools. It is broader than
[`money4band-community-catalog`](https://github.com/MRColorR/money4band-community-catalog),
which only lists M4B-runnable apps.

Promote entries between repos by plain file move — the schema is
identical.

## Referrals

Pool links under `referrals[]` with your contributor handle:

```jsonc
"referrals": [
  { "url": "https://example.com/ref/YOU", "contributor": "YourHandle",
    "added": "2026-04-24" }
]
```

Weighted rotation is driven by shared `scripts/compute_weights.py`
logic across all three M4B repos.
